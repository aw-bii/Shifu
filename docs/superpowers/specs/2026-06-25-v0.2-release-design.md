# MyRA v0.2 Release Design

**Date:** 2026-06-25
**Status:** Approved

## Overview

v0.2 is the first major quality release after the v0.1 foundation. All implementation work across six improvement plans is complete. This spec covers what remains: committing the plan artefacts, bumping the version, writing the user-facing changelog, tagging the release, and producing the binary installer.

## What Was Shipped in v0.2

Six improvement plans were implemented in the period leading up to this release:

### Bug Fixes (audit-fixes + app-flow-audit-fixes)
- **SecurityDialog focus trap re-fires** on queued events via a `trigger` param in `useFocusTrap`
- **SettingsPanel IPC failures** handled gracefully — partial failures no longer blank the panel
- **Double aria-live announcement** during streaming eliminated; `StreamingAnnouncer` is now the single announcement source
- **Pipeline BrowserWindow stale reference** fixed — re-fetched per step to survive window reloads
- **ConvList setState after unmount** guarded with `isMounted` ref
- **AUTH_COMMANDS** extracted into a shared constant; BackendSwitcher auth warning positioned absolutely to avoid toolbar height growth
- **Wizard step 3 subtitle** hidden when no tools need auth
- **BackendSwitcher** warns inline when selected backend is installed but not authenticated
- **Right panels** (Personas, Pipelines, Settings) now animate via `max-width` so the chat area fills available space when panels are closed; close buttons added to Personas and Pipelines panels
- **Wizard install lines** now carry `backend` id so the renderer can route per-backend

### Performance (optimize-performance)
- **ConvList search** debounced at 300ms — IPC fires once per pause, not per keystroke
- **SettingsPanel IPC calls** batched into a single `Promise.all` on mount (4 round-trips → 1)
- **MessageBubble** memoized by `id + content`; attachment cache capped at 100 entries with LRU eviction
- **MessageList** wrapped in `React.memo` to skip re-renders on unrelated parent updates
- **AdapterManager** parallelizes `isAvailable()` and `checkAuth()` per adapter

### Accessibility (harden-accessibility)
- **Toolbar buttons** all have `aria-label` and `aria-pressed`
- **MessageBubble** uses `role="article"` with `aria-label`; timestamp wrapped in `<time dateTime="...">`
- **ConvItem** supports F2 to enter rename mode
- **InputBar** Send/Stop uses conditional render — only one button in the DOM at a time (WCAG 4.1.2)
- **Global focus-visible ring** added via `:focus-visible` CSS rule
- **Skip-to-main-content link** at top of App
- **ConvList** search input has `aria-label`; streaming assistant responses announced via `role="status"` live regions
- **SecurityDialog** has a proper focus trap (Tab/Shift+Tab cycles within dialog, first element auto-focused)
- **Pipeline tabs** use `role="tablist"` / `role="tab"` / `aria-selected` / `aria-controls`
- **PersonaPanel** interactive items converted from `div role="button"` to `<button>`; action buttons extracted to avoid invalid button nesting

### Design System (colorize-design-tokens)
- **CSS custom properties** defined for all semantic colors (`--c-primary`, `--c-bubble`, `--c-surface`, etc.)
- **Tailwind semantic tokens** registered (`bg-primary`, `bg-bubble`, `text-text-muted`, etc.) — dark mode handled by CSS vars, not `dark:` class variants
- **All components** updated to use semantic tokens — hardcoded `bg-blue-600 dark:bg-blue-X` pairs collapsed into single token classes

### Responsive (adapt-responsive)
- **Sidebar panels** clamp to `min(Npx, 80vw)` — never overflow on narrow viewports
- **Toolbar** wraps to a second row on narrow viewports instead of horizontal scrolling
- **ConvItem** supports 600ms long-press to trigger rename on touch devices
- **InputBar textarea** caps at `min(10rem, 40vh)` — scales down on mobile
- **Touch targets** meet 44px minimum on coarse-pointer devices via `.touch-target` class

## Release Finalization

### What the plan produces
1. Six plan documents committed to `docs/superpowers/plans/`
2. Version bump: `0.1.0 → 0.2.0` in `package.json`
3. `CHANGELOG.md` at repo root with user-facing bullet summary
4. Git tag: `v0.2.0`
5. Binary installer via `npm run dist`

### CHANGELOG format
User-facing, grouped by theme: Bug Fixes, Performance, Accessibility, Design System, Responsive. No internal implementation details — describe observable behavior changes only.

### Version bump scope
`package.json` only. No other files contain a hardcoded version string that needs updating.

### Packaging
`npm run dist` invokes `electron-builder` to produce a Windows installer. Output lands in `dist/`. The plan verifies the build succeeds before tagging.

### Git tag
Annotated tag `v0.2.0` on the final commit (after CHANGELOG + version bump). Tag message: `MyRA v0.2.0`.

## Success Criteria

- `npm test` passes (238 tests, all green)
- `npm run build` succeeds with no TypeScript errors
- `npm run dist` produces an installer in `dist/`
- `git tag v0.2.0` exists on the commit that bumps the version
- `CHANGELOG.md` exists at repo root describing v0.2 changes
- Plan docs committed to `docs/superpowers/plans/`
